The Mitnick Attack Lab

Introduction

1994年，Mitnick利用TCP协议中的漏洞和Shimomura的两台计算机之间的信任关系，成功对计算机发起了攻击，这是一种特殊类型的TCP会话劫持。本实验的目的是重现经典的Mitnick攻击，以获得此类攻击的第一手经验。
本实验模拟最初在Shimomura的计算机上的设置，然后发动Mitnick攻击，在Shimomura的两台计算机之间创建一个伪造的TCP会话。如果攻击成功了，我们就能在Shimomural的电脑上执行任何命令。

How the Mitnick Attack Works

Mitnick攻击是TCP会话劫持攻击的一种特例。Mitnick攻击不是劫持受害者A和B之间现有的TCP连接，而是首先代表他们在A和B之间创建一个TCP连接，然后自然地劫持该连接。
在实际的米特尼克攻击中，主机A被称为x终端，也就是攻击目标。米特尼克想登录x终端并在上面运行他的命令。主机B是一个受信任的服务器，允许它不需要密码就可以登录X-Terminal。为了登录X-Terminal, Mitnick必须模拟可信服务器，所以他不需要提供任何密码。

这种攻击共四个步骤：

  1. 序列号预测。在攻击之前，Mitnick需要了解X-terminal上的初始序列号(ISN)的模式(在那个年代，ISN不是随机的)。Mitnick向X-terminal发送SYN请求，收到SYN+ACK响应，然后向X-terminal发送RESET包，从X-terminal的队列中清除半开连接(防止队列被填满)。重复了二十次之后。他发现两个连续的TCP ISN之间存在一种模式。这使得米特尼克能够预测ISN，这对攻击至关重要。
  2. SYN flood攻击可信服务器。为了从可信服务器向X-Terminal发送连接请求，Mitnick需要从可信服务器向X-Terminal发送SYN包。X-Terminal将响应一个SYN+ACK包，该包被发送到受信任的服务器。由于受信任的服务器实际上并没有发起请求，所以它会向X-Terminal发送一个RESET包，要求X-Terminal停止3次握手。这种行为给米特尼克的袭击带来了麻烦。

为了解决这个问题。米特尼克不得不关闭可信服务器。因此，在欺骗之前，Mitnick对服务器发起了SYN泛洪攻击。那时，操作系统更容易受到SYN泛洪攻击。攻击实际上可以关闭受信任的计算机，使其完全沉默。

  1. 欺骗TCP连接。Mitnick想使用rsh(远程shell)在X-Terminal上运行一个后门命令;一旦设置了后门，他就可以登录X-Terminal。要在X-Terminal上运行远程shell, Mitnick需要通过身份验证，即他需要在X-Terminal上拥有一个有效的帐户并知道其密码。

Shimomura经常需要从可信服务器登录到X-Terminal。为了避免每次都输入密码，他在X-Terminal的.rhosts文件中添加了一些信息，这样当他从受信任的服务器登录到X-Terminal时，就不会被要求输入密码。米特尼克想利用这种相互信任的关系。他需要在可信服务器和X-Terminal之间创建一个TCP连接，然后在这个连接中运行rsh。他首先向X-Terminal发送一个SYN请求，使用可信服务器的IP作为源IP地址。然后X-Terminal向服务器发送SYN+ACK响应。由于服务器已经关闭，它不会发送RESET来关闭连接。
为了完成三次握手协议，Mitnick需要欺骗一个ACK数据包，该数据包必须确认X-Terminal的SYN+ACK数据包中的序列号。不幸的是，SYN+ACK响应只发送给受信任的服务器，而没有发送给Mitnick，他看不到序列号。然而，由于先前的调查，Mitnick能够预测这个数字是什么，因此他能够成功地欺骗发送给X-Terminal的ACK响应，以完成TCP三次握手。

  1. 运行远程shell。使用可信服务器和X-Terminal之间建立的TCP连接，Mitnick可以向X-Terminal发送远程shell请求，要求它运行命令。使用这个命令，Mitnick想在X-Terminal上创建一个后门，这样他就可以随时在X-Terminal上获得shell，而不必重复攻击。

他所需要做的就是在X-Terminal的.rhosts文件中添加“+ +”。他可以通过在X-Terminal上使用rsh执行以下命令来实现:”echo + + > .rhosts”。由于rsh和rlogin程序使用.rhosts文件进行身份验证，因此X-Terminal将信任来自任何人的每个rsh和rlogin请求。

Lab Environment Setup

在本实验中，我们需要三台机器，一台用于X-Terminal，一台用于可信服务器，另一台用于攻击者。在真正的Mitnick攻击中，攻击者是一台远程机器。在本实验中，为了简单起见，我们将这三台机器放在同一个网络上。
进入实验目录，进入Labsetup文件夹，使用docker-compose build构建docker映像，然后使用docker-compose up启动容器。即可看到三个虚拟机在运行。

Configuration

rsh服务器程序使用两个文件进行身份验证: .rhosts和/etc/hosts.equiv。每次服务器接收到远程命令请求时，它都会检查/etc/hosts.equiv。如果请求来自存储在文件中的主机名，服务器将接受它而不要求输入密码。如果/etc/hosts.Equiv不存在或没有该主机名，RSH将检查用户主目录上的.rhosts文件。
Shimomura经常需要从可信服务器在X-Terminal上运行远程命令。为了避免输入密码，他在X-Terminal主机上创建了一个.rhosts文件，并将可信服务器的IP地址放入该文件中。注意，.rhosts文件必须位于用户主目录的顶层，并且只能由所有者/用户写入。

Task 1: Simulated SYN flooding

Mitnick攻击时的操作系统很容易受到SYN泛洪攻击，这种攻击可以使目标机器静音，甚至关闭它。虽然SYN泛洪攻击已经不会对现代操作系统造成这样的损害，但是我们可以模拟这种效果。
我们可以手动停止受信任的服务器容器，但这还不够。当X-Terminal接收到来自受信任服务器的SYN报文时，将返回SYN+ACK报文。在发送此报文之前，需要知道受信任服务器的MAC地址。首先检查ARP缓存。如果没有可信服务器的表项，X-Terminal将发送ARP请求报文请求MAC地址。由于受信任的服务器已经静音，没有人会回应ARP请求，因此X-Terminal无法发出响应。导致TCP连接无法建立。
在实际攻击中，可信服务器的MAC地址实际上在X-Terminal的ARP缓存中。即使不是，在使受信任的服务器静默之前，我们也可以简单地欺骗X-Terminal一个来自受信任服务器的ICMP回显请求，这将触发X-Terminal回复可信服务器，从而获得可信服务器的MAC地址，并将其保存到缓存中。

Task 2: Spoof TCP Connections and rsh Sessions

现在我们已经“关闭”了受信任的服务器，我们可以模拟受信任的服务器，并尝试使用X-Terminal启动一个rsh会话。由于rsh运行在TCP之上，我们首先需要在可信服务器和X-Terminal之间建立一个TCP连接，然后在这个TCP连接中运行rsh。
Mitnick攻击的难点之一是TCP序列号的预测。在TCP序列号不是随机化的时候，这是可能的。然而，现代操作系统现在将其TCP序列号随机化(作为对抗TCP会话劫持攻击的对策)，因此预测数字变得不可行。为了模拟原始Mitnick攻击的情况，我们可以使用Wireshark嗅探数据包，这样我们就可以获得序列号，而不是猜测它们。
限制。为了尽可能接近地模拟原始Mitnick攻击，我们只能使用捕获数据包的以下字段：TCP序列号字段、TCP标志字段、所有长度字段。

第一个TCP连接是由攻击者通过欺骗的SYN包发起的。在X-Terminal接收到SYN包后，它将依次向受信任的服务器发送SYN+ACK包。因为服务器已经关闭，所以它不会重置连接。位于同一网络上的攻击者可以嗅探数据包并获得序列号。

Task 2.2: Spoof the Second TCP Connection

建立第一个连接后，X-Terminal将发起第二个连接。rshd使用这个连接发送错误消息。在我们的攻击中，我们不会使用这个连接，但是如果没有建立这个连接，rshd将停止而不执行我们的命令。因此，我们需要使用欺骗来帮助X-Terminal和受信任的服务器完成建立此连接。

我们需要编写另一个嗅探和欺骗程序，嗅探发送到受信任服务器的9090端口的TCP流量(假设任务2.1中使用9090)。当它看到一个SYN包时，它应该响应一个SYN+ACK包。响应的包应该有一个任意32位无符号整数的初始seq值，随机生成也行，但是为了方便这里直接使用了对方发过来的序列号tcp.seq = pkt[TCP].seq。如果两个连接都建立成功，rshd将执行rsh数据包中包含的命令。

Task 3: Set Up a Backdoor

在Task 2中，我们在攻击中只运行了一个touch命令，以证明我们可以成功地在X-Terminal上运行一个命令。如果我们以后想要运行更多的命令，我们总是可以发起相同的攻击。这很不方便。
为了不让我们重复发起攻击，我们在X-Terminal中设置一个后门。这个后门允许他在任何时候正常登录X-Terminal，而不需要输入任何密码。为了实现这个目标，正如我们前面说到，我们所需要的就是将字符串“+ +”添加到.rhosts文件中。
这个设置很简单，我们只需要修改task2.1第三部分的代码的touch命令部分，修改为echo + + > .rhosts即可。

wireshark中可以看到，我们成功设置remote shell为echo + + > .rhosts，并且我们使用seed-attacker已经可以直接免密远程登录到X-Terminal。
