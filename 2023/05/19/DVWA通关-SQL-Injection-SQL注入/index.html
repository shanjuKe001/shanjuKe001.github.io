前言

    在查询数据库中的表的时候出现了错误，如下图所示。

    所幸在网上找到了解决答案，是编码问题导致的。因为information_schema数据库里面的编码方式是utf8_general_ci，而DVWA中user表的编码方式是utf8_unicode_ci，我们将网站中的编码修改为utf8_general_ci即可，为了防止其他地方继续出现该问题，我们将网站的所有编码修改为utf8_general_ci。
    首先，我们打开phpstudy，选择软件管理->网站程序，安装phpMyAdmin，然后点击管理。

    然后我们进入到网站，左侧选择dvwa，然后点击右上方的操作，滑到底，更改排序规则为utf8_general_ci，勾选更改所有表排序规则，最后点击执行即可。

    然后刷新界面，我们就可以查询到数据库中的表和字段等信息。


Medium

源码分析

Medium级别的代码相对于Low，首先是由GET类型的传参改为POST类型，改变了闭合方式；另外界面变成了下拉菜单，已经无法通过输入传参；然后，使用了mysqli_real_escape_string函数对特殊字符进行了转义。具体如下图所示：

SQL注入

High

源码分析

High中新增了SESSION新建一个界面，从SESSION中获取id的值，使用单引号闭合。这样我们无法直接从当前页面注入，但是可以从新界面注入，貌似没什么用。当然，在id后面新增了一个LIMIT 1，只提供一个参数，用来限制输入词数，但是可以直接注释掉。

SQL注入

事实上我们可以采取和Low级别相同的解法，只是换了一个输入界面而已。。。

在输入框输入和Low级别时一样的语句即可成功注入。

当然，我们同样可以像Medium级别一样，使用Burp Suite进行抓包，修改参数，不过因为有一个session，我们需要先forward一次，然后修改第二个包即可。在此就不做示范了。

Impossible

源码分析

Impossible级别的代码采用了PDO技术，使得代码和数据库之间不再有交互的可能，能够有效防止SQL注入；另外，还对返回结果进行了判断，只有返回的查询结果数量为1时才会成功输出；而且代码中还加入了Anti-CSRFtoken机制，进一步提高了安全性。至于什么是PDO技术，以及它是如何防止SQL注入的，大家可以到网上自行查找，此外，我的参考链接里面也有提供。
